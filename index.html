Here is a write-up on my final assignment:

Introduction
The game I created for my final assignment is called Survival Instincts. The game involves the player who will navigating through the skies on a helicopter with his/her friend. The first segment of the game involves an animated story that narrates how a storm caused the helicopter to crash. After which, the player blacks out and upon gaining consciousness, realizes that his/her friend had disappeared. The player then has to play the game which involves killing poisonous, exotic flies before it reaches the player at the bottom of the screen. The game ends after the player has stayed alive for 20 seconds, or when the health meter decreases to zero. I decided to create two segments for my assignment – animated story and the game, as I wanted to challenge my skills as well as to be able to make use for the variety of skills that we have learnt throughout the module.

Process
I started off with the HTML layout that I had used for my previous assignment and began modifying it to fit my game. I made use of Javascript library, Raphael.js to create graphics and animation for my game. For instance, I could create rectangles, circles and texts, and tweak the positioning and attributes of these graphics to fit the game. Raphael animation was also used to animate the helicopter and lightning.

I made use of arrays and loops for the rainstorm and flies. For the rainstorm, I created an array of raindrops and looped them for 150 (maxRaindrop) of raindrops. I created a draw function for enabling the raindrops to move by changing its x and y coordinates, and called on that function every 40ms with setInterval. I also referred to the emitter tutorial that we had in class, to enable more raindrops to be created even after 150 (maxRaindrop) raindrops had been created. The “%” operator ensures that the value of i will not exceed 150, hence, the loop will not stop. Similarly, this method was used for creating and looping the array of flies for the fly game. The map and distance functions that we used for assignment 4 came in handy for enabling me to scale the distance for the x and y coordinates and rates for the fly array.

As there were many elements in final assignment, I had to organize my code neatly to avoid confusion. The placement of the variables was also very important as I met with many error messages when variables were not defined globally. Using console messages helped greatly in troubleshooting the errors that I encountered. I realized that it was very important to think calmly and logically about the origin of the error.

If statements were very useful in enabling me to run functions when certain conditions were met. For instance, if the mousedown coordinates is within 100, the fly will be set off-screen and a splat sound will be played. If the fly is very close to the bottom of the screen, a panic alert will sound and some exclamations marks will appear at the bottom right hand corner. If the health value is zero, it will run the Game Over function. If the time runs out and the player is still alive, it will run Survived function, which informs the user that s/he has found his/her friend and civilization. The if statements also enabled me to set the difficulty of the game depending on the selected difficulty level.

Referring to coding platforms such as w3schools.com aided me in improving and trying to find solutions for my game. I was initially trying to create a health bar using a Raphael graphic, but after searching online (https://css-tricks.com/html5-progress-element/), I realized that there is a HTML progress bar element which can be used for health bars in games. Hence, I decided to utilize it and place it outside of my Raphael “paper” to avoid overlapping with the game.

After I had conceptualized the theme and mechanics for my game, I began to sourced for suitable images and sound files for my game to enhance the game experience for the user. All the sound files were sourced from Freesound.org, and I modified the length of the sounds using audio software to fit the game. Timers and event listeners enabled me to play the audios at the appropriate time. Through researching online, I discovered how to make use of a function to gradually decrease volume, so that there is a better transition to the next page.

Further Improvements

In order to enable users to skip the story animation scenes, two buttons were created beside the mySVGCanvas Div. The first button enables the user to skip the storyline, regardless of which scene they are currently in. The drawback of this is that some animation may not have appeared on the screen when the button is clicked, due to setTimeout functions on the scene text and next buttons. To overcome this problem, I included a setTimeout inside of the eventListener to hide all the elements, in case new elements appear after the button was clicked. Another drawback is that error codes will appear if the button is clicked in the first scene, before it starts raining. This is because the rain array is only initialized inside of var scene2. To counter this problem, I created another button is specifically for stopping the rain. The event listener listens for clicks, and will run a function to hide the rain if the rain has started.

During the coding of my game, I faced many challenges and error codes. In particular, I had great difficulty in trying to hide and show the appropriate elements at the correct time. Despite many attempts, I was unable to hide all the flies off the screen after the game ends. The flies that are already on the screen will still stay stationary on that screen after the drawFly interval is cleared. Also, restarting the game works if it was “game over”- new flies will appear. However, if the player survived the game and wanted to replay, new flies do not appear. I regret not being able to fix this bug, but I suspect it may be because the survived function was created outside of the function containing the initialization of the fly array.

To further enhance the game experience, perhaps obstacles could be added into the game to increase difficulty. For instance, clicking on obstacles such as endangered animal species, could result in loss of health, penalty points or an extension of time (i.e having to survive in a longer period of time). Overall, while there is still room for improvement, the coding process was very valuable and challenged my limits.
